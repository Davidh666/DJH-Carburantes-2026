2. Flujo de Venta Controlada

Flujo Operativo Paso a Paso

1️⃣ Cliente llega a bomba
→
2️⃣ Operador inicia sesión
→
3️⃣ Ingresa placa
4️⃣ Sistema busca cliente
→
5️⃣ Calcula promedio semanal
→
6️⃣ Determina límite
7️⃣ Valida cantidad
→
✅ Si está dentro del límite → AUTORIZAR
❌ Si excede el límite → BLOQUEAR

3. Algoritmo de Cálculo de Cupo

Promedio Semanal (P_s)

P_s = SUM(litros vendidos en últimos 28 días) / 4 semanas

Límite = P_s × (1 + factor_holgura)

Ejemplo con factor_holgura = 0.10 (10%):
- P_s = 100L → Límite = 100 × 1.10 = 110L

Casos Especiales

Cliente Nuevo (sin historial):
P_s = 0
Límite = cupo_base_inicial (configurado en empresa)
Ejemplo: Si cupo_base_inicial = 120L → Límite = 120L
Cliente Suspendido:
❌ Estado = "Suspendido"
→ Transacción rechazada automáticamente

4. Escenarios de Validación

Escenario A: Cliente Nuevo

Entrada: placa = "ABC123", litros_solicitados = 150 Procesamiento: 1. SELECT cliente WHERE placa = "ABC123" → NO EXISTE 2. INSERT cliente (documento, placa, ..., estado="Activo") 3. P_s = 0 (sin historial) 4. Límite = 120L (cupo_base_inicial) 5. 150L > 120L → BLOQUEADO Salida: { estado: "bloqueado", límite: 120, promedio_semanal: 0 }

Escenario B: Cliente con Historial dentro del Límite

Entrada: placa = "ABC123", litros_solicitados = 90 Procesamiento: 1. SELECT cliente WHERE placa = "ABC123" → EXISTE 2. SELECT SUM(litros) FROM venta WHERE cliente_id=1 AND fecha >= (hoy-28) → Total = 400L 3. P_s = 400L / 4 = 100L 4. Límite = 100 × (1 + 0.10) = 110L 5. 90L ≤ 110L → AUTORIZADO 6. UPDATE tanque SET stock_actual = stock_actual - 90 7. INSERT venta (...) Salida: { estado: "autorizado", litros: 90, límite: 110, total: 765000 }

Escenario C: Cliente intenta exceder límite

Entrada: placa = "ABC123", litros_solicitados = 150 Procesamiento: 1. SELECT cliente WHERE placa = "ABC123" → EXISTE 2. P_s = 100L (calculado) 3. Límite = 110L 4. 150L > 110L → BLOQUEADO Salida: { estado: "bloqueado", mensaje: "La cantidad excede el límite semanal", límite: 110, promedio_semanal: 100 }
